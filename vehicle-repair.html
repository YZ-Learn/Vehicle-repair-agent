车辆维修智能助手

面向维修工程师的 AI 助手——输入故障现象或故障码，自动检索知识库、调用工具，输出诊断建议与维修方案。

LangGraph DeepSeek RAG Chroma Streamlit SQLite

整体架构

前端 Streamlit 聊天界面 · 实时流式输出 · 历史记录 · 用户反馈
Agent 自研轻量多步推理引擎 · 解析意图 → 检索 → 调工具 → 综合回答 · 最大 8 步迭代
工具层 5 个注册工具 · JSON Schema 描述 · LLM 自动判断调用时机
RAG Chroma 向量库 · Query 改写 + Rerank 重排序 · 双通道检索增强
模型层 抽象工厂模式 · YAML 配置驱动 · 支持 DeepSeek / 通义千问 / 本地 Embedding 无感切换
存储 SQLite 对话持久化 · 记录 Token 用量 / 工具调用链 / 参考文档
工作流程： 用户提问 → RAG 检索知识库 → Agent 分析意图 → 调用工具（故障码 / 配件 / 工时） → 综合知识库 + 工具结果 → 流式输出回答

工具集

search_trouble_code
查询故障码（DTC）含义与诊断建议
参数: code (str)
search_part_info
查配件 OE 号、价格区间、常见故障
参数: part_name (str)
get_safety_warning
获取特定维修操作的安全警告
参数: operation (str)
estimate_repair_time
工时估算（按维修项目 + 车型）
参数: repair_item, vehicle_type
get_current_time
获取当前日期时间
参数: 无

知识库

6 篇车辆维修文档，覆盖核心系统

发动机系统 制动系统 电气系统 变速箱系统 空调系统 底盘悬挂系统

每篇按「故障现象 → 可能原因 → 诊断步骤 → 维修建议」结构化组织 · 即改即用，无需改代码

设计亮点

  • 自研 Agent 引擎 — 不依赖 LangChain 框架，轻量可控。ReAct 风格多步推理，最大 8 步迭代，错误自动兜底。
  • 双通道检索增强 — Query 改写 + Rerank 重排序，知识召回精度明显优于直接向量检索。
  • 抽象模型工厂 — YAML 一键切换模型提供商（DeepSeek / 通义千问），支持本地 Embedding 离线运行。
  • @tool 装饰器注册 — 新增工具只需一个装饰器 + 一个函数，自动生成 OpenAI 兼容的 JSON Schema。
  • 全链路可观测 — 每次对话记录：Token 用量、工具调用链、参考文档、用户反馈，数据不丢。
  • 零配置扩展 — 换行业只需替换 knowledge/ 目录 + 改工具名，无需修改框架代码。
